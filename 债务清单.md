## 技术债务清单

---

记录已知、尚未处理的改进项，按优先级与影响范围维护。完成某项后从本清单移除，并在对应 PR/提交说明中简要注明。

### 性能与架构

| 项 | 影响 | 建议方向 | 相关位置 |
|----|------|----------|----------|
| **主布局嵌套过深** | 全页 `NestedScrollView` 包裹多层 `ConstraintLayout`，离屏子树仍参与测量；`EditText` 与外层滚动叠加，易出现滑动冲突与过度绘制 | 非必要容器改为 `merge` 或扁平 `ConstraintLayout`；为编辑区固定高度或独立滚动区域；减少 background + elevation 叠加 | `app/src/main/res/layout/activity_main.xml` |
| **大文本经 Intent 往返拷贝** | 主界面与文本编辑器之间通过 `Intent` extra 传递全文；接近 TTS 最大输入长度时内存占用与页面跳转延迟明显 | 使用临时文件、`SharedMemory` 或进程内会话缓存，Intent 仅传 URI 或 revision id | `MainActivity.java`（启动编辑器、接收结果）；`TextEditorActivity.java`（`EXTRA_TEXT`） |
| **TTS 浏览器双层可滚动容器** | 语言页为 `ListView` 嵌在 `ViewPager2` 的 `RecyclerView` 中，快速滑动长列表时测量与触摸协调成本高 | 改为单层 `RecyclerView`（如 `ConcatAdapter`），或 Tab 内独立 `Fragment`；必要时启用嵌套滚动协调 | `TtsBrowserActivity.java`；`fragment_tts_browser.xml`、`fragment_tts_browser_with_search.xml` |
| **双 TTS 实例** | 主界面与 TTS 浏览器同时打开时各持有一套 `TextToSpeech`，内存与引擎连接加倍 | 浏览器只读模式复用 Application 级仓库或统一生命周期管理；明确单例归属与释放时机 | `MainActivity.java`；`TtsBrowserActivity.java` |
| **主界面 `updateStatusInfo` 粒度粗** | 单次调用刷新引擎信息、保存目录路径、调试区可见性、朗读状态等整块 UI；无关字段变更也会触发路径解码与布局重绑 | 拆分为按需增量更新（如目录变更只刷新目录块、引擎切换只刷新引擎块） | `MainActivity.java`（`updateStatusInfo` 及各 `update*Visibility` 调用方） |
| **语言目录加载时重建 `LanguageAdapter`** | `applyTtsCatalog` 每次 `new LanguageAdapter` 并 `setAdapter`；目录较大时初始化与引擎切换后重建下拉成本偏高 | 与发音人列表相同，提供 `updateData()` 复用适配器实例 | `MainActivity.java`（`applyTtsCatalog`、`LanguageAdapter`） |

### 主界面 TTS 与控件

| 项 | 影响 | 建议方向 | 相关位置 |
|----|------|----------|----------|
| **TTS 状态刷新仍依赖轮询** | 朗读/保存/合成/拷贝进行中每 300ms 刷新按钮与状态文案；空闲时已停止轮询，但仍非纯事件驱动 | 以 `UtteranceProgressListener` 与状态机事件完全替代 `Handler.postDelayed` 轮询 | `MainActivity.java`；`Constants.TTS_STATUS_UPDATE_INTERVAL`；`TtsSpeakStatusHelper.java` |
| **换语言时发音人下拉全量 `notifyDataSetChanged`** | 已复用 `VoiceAdapter` 实例，但 `refreshAfterVoiceListChange` 仍全量刷新下拉项；发音人很多时切换语言仍有卡顿风险 | 差异化刷新（`notifyDataSetInvalidated` 限制展开时绑定），或下拉展开后再异步填充 | `MainActivity.java`（`updateVoiceList`、`VoiceAdapter`） |
| **语速/音高滑块拖动不写 TTS** | 拖滑块仅更新 UI 与内存变量；±、重置、朗读/保存前才调用 `setSpeechRate`/`setPitch`；朗读中拖滑块无法实时听感 | 若产品需要：在 `onStopTrackingTouch` 写入，或朗读中对 IPC 做节流 | `MainActivity.java`（`SeekBarHelper` 监听器） |
| **输入框与外层滚动手势协调复杂** | `ManualScrollEditText` + `NestedScrollView` + `dispatchTouchEvent` 失焦逻辑叠加，边界场景多、维护成本高 | 与「主布局嵌套过深」一并治理：固定编辑区高度或独立滚动容器，简化触摸分发 | `ManualScrollEditText.java`；`MainActivity.java`；`activity_main.xml` |

### TTS 浏览器

| 项 | 影响 | 建议方向 | 相关位置 |
|----|------|----------|----------|
| **搜索过滤仍全量 `notifyDataSetChanged`** | 已 200ms 防抖，过滤完成后仍重建整个 `LanguageVoiceAdapter` 可见数据集 | 使用 `DiffUtil`、过滤结果缓存或分段索引，减少长列表刷新开销 | `TtsBrowserActivity.java`（`LanguageVoiceAdapter.filter`） |
| **引擎页 `refreshData` 全量 `notifyDataSetChanged`** | 切换引擎或刷新数据时清空适配器缓存并通知 `ViewPager2` 全部页重绑 | 按页差异化刷新，或 `ViewPager2` 子页用 `Fragment` 独立生命周期 | `TtsBrowserActivity.java`（`TtsBrowserPagerAdapter.refreshData`） |

### 次要优化（按需处理）

| 项 | 影响 | 建议方向 | 相关位置 |
|----|------|----------|----------|
| **合成完成后立即关闭 PFD（API 30+）** | `synthesizeTextToUri` 在 API 30+ 上于调用后立即 `pfd.close()`，可能影响异步合成正确性 | 在 utterance 完成回调中关闭 `ParcelFileDescriptor` | `MainActivity.java`（`synthesizeTextToUri`） |
| **`ToastHelper` 静态持有 Toast 视图** | 可能延长 Activity 相关视图链的存活窗口 | 使用 `applicationContext` 创建 Toast，或改为 `WeakReference` | `ToastHelper.java` |
| **`DelayedTaskHelper` 任务不可取消** | 清空按钮等延迟任务在 Activity 销毁后仍可能短暂执行 | 返回可 `removeCallbacks` 的 token；`onDestroy` 清理 | `DelayedTaskHelper.java`；`ClearButtonHelper.java` |
| **`ButtonTextHelper` 文本自适应测量** | 按钮文案变化时 `post` 后循环 `measureText` 缩小字号；多按钮同时更新时有主线程测量开销 | 优先使用系统 `autoSizeTextType`；或限定仅在布局完成后测量一次 | `ButtonTextHelper.java` |
